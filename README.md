# PropManager (Property Management System)

ระบบจัดการอสังหาริมทรัพย์แบบครบวงจรสำหรับผู้จัดการโครงการ (Manager) และผู้อยู่อาศัย (Resident) ครอบคลุมงานสำคัญตั้งแต่การจัดการทรัพย์สิน สัญญาเช่า บิล/การชำระเงิน งานซ่อมบำรุง แชตแบบเรียลไทม์ ประกาศข่าวสาร ไปจนถึงการแจ้งเตือนแบบ Push Notification

---

## โปรเจกต์นี้คืออะไร

`PropManager` คือแพลตฟอร์มบริหารที่พัก/โครงการเช่า ที่ช่วยให้:

- ผู้จัดการ: ดูแลโครงการ ห้องเช่า ผู้เช่า สัญญา บิล และคำร้องต่าง ๆ ได้ในระบบเดียว
- ผู้อยู่อาศัย: ดูสัญญา บิล ประวัติชำระเงิน แจ้งซ่อม ส่งข้อความ และขอสินทรัพย์ได้สะดวก
- ทั้งสองฝั่ง: สื่อสารและติดตามสถานะได้แบบเรียลไทม์ ลดงานซ้ำซ้อนและความผิดพลาด

เหมาะสำหรับอพาร์ตเมนต์ คอนโด หอพัก บ้านเช่า และโครงการขนาดเล็กถึงกลางที่ต้องการระบบดิจิทัลครบชุด

---

## ระบบทำอะไรได้บ้าง (What it can do)

### ฝั่งผู้จัดการ (Manager)

- แดชบอร์ดรวมภาพรวมธุรกิจ (สัญญาใช้งานอยู่, งานซ่อมคงค้าง, รายการชำระเงินรอตรวจ)
- จัดการทรัพย์สิน/โครงการ (Properties) พร้อมข้อมูลหน่วย ห้อง ราคา สถานะ การเข้าพัก
- จัดการผู้พักอาศัย (Residents) พร้อมค้นหา/กรอง/แบ่งหน้า
- จัดการสัญญาเช่า (Leases) และสถานะสัญญา
- ออกบิล (Invoices/Billing) พร้อมค่าน้ำ-ค่าไฟจากมิเตอร์เริ่ม/สิ้นสุด และค่าบริการอื่น
- ตรวจสอบการชำระเงิน (Payments) และหลักฐานการโอน
- จัดการงานซ่อมบำรุง (Maintenance) พร้อมลำดับความสำคัญและสถานะ
- สื่อสารผ่านแชตแบบเรียลไทม์ (Messages)
- สร้างประกาศข่าวสาร (Announcements)
- จัดการทรัพย์สินประกอบ (Assets) และคำขอใช้งาน (Asset Requests)
- จำกัดจำนวนการใช้งานตามแพ็กเกจ (Plan limits)

### ฝั่งผู้อยู่อาศัย (Resident)

- ดูแดชบอร์ดส่วนตัวและข้อมูลที่พัก
- ดู/ยอมรับสัญญาเช่า และอัปโหลดเอกสารยืนยันตัวตน
- ดูบิลคงค้าง รายละเอียดค่าน้ำ-ไฟ และกำหนดชำระ
- ดูประวัติการชำระเงิน
- แจ้งซ่อม ติดตามสถานะ และคุยกับผู้ดูแลแบบเรียลไทม์
- ใช้งานระบบแชตกับผู้จัดการ
- ดูสินทรัพย์ที่ใช้งานได้ และส่งคำขออนุมัติ

### ฟีเจอร์ร่วมสำคัญ

- แจ้งเตือนในระบบ (In-app notifications)
- Push Notification ผ่าน Service Worker + VAPID
- รองรับ 2 ภาษา: ไทย/อังกฤษ
- รองรับ Dark Mode
- แสดงกราฟวิเคราะห์ด้วย Chart.js

## ระบบทำงานอย่างไร (How it works)

สถาปัตยกรรมเป็นแบบ Monorepo แยก Frontend และ Backend ชัดเจน

```text
Property-Managemt/
├─ packages/
│  ├─ app/      # Nuxt 3 (Frontend)
│  └─ server/   # Strapi 5 (Backend API + CMS)
└─ package.json # สคริปต์รวมระดับโปรเจกต์
```

### 1) Frontend (`packages/app`)

- พัฒนาโดย Nuxt 3 + Vue 3 + Tailwind CSS
- เป็น Dashboard แบบ authenticated app (ปิด SSR)
- เรียก API จาก Strapi ผ่าน `runtimeConfig.public.strapiUrl`
- จัดการสถานะล็อกอินผ่าน `useAuth()` และ `localStorage`
- รองรับ Socket.IO client เพื่ออัปเดตข้อความ/สถานะทันที

### 2) Backend (`packages/server`)

- พัฒนาโดย Strapi 5 (Headless CMS)
- ให้ REST API สำหรับข้อมูลธุรกิจทั้งหมด เช่น property, lease, billing, payment ฯลฯ
- ใช้ `users-permissions` สำหรับ Authentication/JWT และ role-based access
- รองรับ Web Push ผ่าน `web-push` และเก็บ subscription
- มีโครงสร้าง content-types สำหรับโมเดลหลักของระบบ

### 3) Real-time และ Notification

- แชตและอัปเดตสถานะสำคัญใช้ Socket.IO
- แจ้งเตือนแบบ push ใช้ VAPID keys + Service Worker (`public/sw.js`)
- มี notification badge แยกตามประเภทเหตุการณ์

## ฟังก์ชันหลักของแต่ละโมดูล (Core Functions)

### การยืนยันตัวตนและสิทธิ์

- ล็อกอิน/สมัครสมาชิกด้วย Strapi Auth
- แยกบทบาทผู้ใช้ `manager` และ `resident`
- Middleware ป้องกันเส้นทางและบังคับเข้าหน้าตามบทบาท

### การจัดการทรัพย์สินและสัญญา

- สร้าง/แก้ไข/ลบข้อมูลโครงการและประเภทห้อง
- จัดการสัญญาเช่าพร้อมสถานะและช่วงเวลาใช้งาน
- ผูกความสัมพันธ์ Resident ↔ Property ↔ Unit Type ↔ Lease

### การเงินและบิล

- ออกใบแจ้งหนี้หลายประเภท (ค่าเช่า, ค่าสาธารณูปโภค, ค่าบริการอื่น)
- คำนวณหน่วยใช้น้ำ-ไฟจากมิเตอร์เริ่มต้น/สิ้นสุด
- ติดตามสถานะชำระเงิน (pending/reviewing/completed/failed ฯลฯ)

### งานซ่อมบำรุงและบริการ

- ส่งคำร้องแจ้งซ่อมพร้อมหมวดหมู่/ความเร่งด่วน/รูปภาพ
- อัปเดตสถานะงานและสื่อสารในเธรดข้อความ
- ฝั่งผู้พักสามารถติดตามความคืบหน้าแบบเรียลไทม์

### การสื่อสารและประกาศ

- แชตส่วนตัวระหว่างผู้จัดการและผู้อยู่อาศัย
- ส่งข้อความพร้อมไฟล์ภาพ
- จัดการประกาศภายในโครงการ (ปักหมุด/จัดหมวด/กำหนดความสำคัญ)

### แพ็กเกจและข้อจำกัดการใช้งาน

- มีโมเดล `Plan` สำหรับกำหนดเพดาน เช่น จำนวนทรัพย์สินสูงสุด/จำนวนยูนิตต่อทรัพย์สิน
- ตรวจสอบข้อจำกัดก่อนสร้างข้อมูลสำคัญ

## เหมาะกับใคร (Who this product suits)

เหมาะกับกลุ่มต่อไปนี้:

- เจ้าของอพาร์ตเมนต์/หอพัก ที่ต้องการลดงานเอกสารและจัดการผู้เช่าเป็นระบบ
- ผู้จัดการคอนโดหรือโครงการเช่าขนาดเล็ก-กลาง ที่ต้องติดตามงานซ่อมและการเงินแบบศูนย์กลาง
- ธุรกิจอสังหาฯ ที่ต้องการช่องทางสื่อสารกับผู้พักแบบทันที
- ทีมที่ต้องการระบบสองบทบาท (Manager/Resident) และขยายต่อได้จาก Strapi

อาจยังไม่เหมาะกับองค์กรที่ต้องการโมดูลบัญชีเชิงลึกระดับ ERP หรือ workflow องค์กรขนาดใหญ่มาก (สามารถต่อยอดเพิ่มได้ในอนาคต)

## เทคโนโลยีที่ใช้ (Tech Stack)

- Frontend: `Nuxt 3`, `Vue 3`, `Tailwind CSS`, `Chart.js`, `Socket.IO Client`, `GSAP`
- Backend: `Strapi 5`, `@strapi/plugin-users-permissions`, `Socket.IO`, `web-push`
- Database: ค่าเริ่มต้นรองรับ `SQLite` (ปรับเป็น MySQL/PostgreSQL ได้)
- Runtime: `Node.js >= 18` (ฝั่ง server package ระบุช่วง `>=20 <=24.x.x`)

## โครงสร้างข้อมูลหลัก (ตัวอย่างโมเดล)

ระบบมี Content Types หลัก เช่น:

- `property` — ข้อมูลโครงการ/ทรัพย์สิน
- `unit-type` — ประเภทห้อง/ยูนิต
- `lease` — สัญญาเช่า
- `billing` — ใบแจ้งหนี้/บิล
- `payment` — ธุรกรรมการชำระเงิน
- `maintenance` และ `maintenance-message` — งานซ่อมและข้อความในงาน
- `conversation` และ `message` — แชต
- `announcement` — ประกาศข่าวสาร
- `asset` และ `asset-request` — สินทรัพย์และคำขอใช้งาน
- `notification` — การแจ้งเตือน
- `plan` — แพ็กเกจและข้อจำกัด
- `push-subscription` — อุปกรณ์ที่สมัครรับ push

## เริ่มต้นใช้งาน (Quick Start)

> หมายเหตุ: โปรเจกต์นี้เป็น monorepo ควรติดตั้ง dependency ทั้ง `app` และ `server` ให้ครบ

### 1) ติดตั้ง dependencies

```bash
npm run install:all
```

### 2) ตั้งค่า Environment

กำหนดไฟล์ `.env` ให้กับฝั่ง backend (`packages/server`) และตั้งค่า URL ฝั่ง frontend (`packages/app`) เช่น:

- `NUXT_PUBLIC_STRAPI_URL` สำหรับ frontend
- `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_EMAIL` สำหรับ push notifications
- ตัวแปรพื้นฐานของ Strapi (`APP_KEYS`, `JWT_SECRET`, `API_TOKEN_SALT` ฯลฯ)

### 3) รันระบบระหว่างพัฒนา

เปิด 2 เทอร์มินัล:

```bash
npm run dev --prefix packages/server
```

```bash
npm run dev --prefix packages/app
```

หรือใช้สคริปต์ระดับ root แยกทีละตัว:

```bash
npm run dev:server
npm run dev:app
```

### 4) Build

```bash
npm run build:server
npm run build:app
```
